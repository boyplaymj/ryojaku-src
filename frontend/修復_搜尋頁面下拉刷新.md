# 🔧 修復：搜尋頁面下拉刷新功能

## 📋 問題描述

**問題：** 在搜尋團局頁面（Search.tsx）下拉頁面無法觸發刷新功能

## 🔍 根本原因

Search.tsx 頁面**沒有註冊下拉刷新的處理函數**。

雖然 `PullToRefresh` 組件包裹了所有頁面，但每個頁面需要使用 `usePullToRefresh` hook 來告訴系統「當用戶下拉時要執行什麼操作」。

### 對比其他頁面：

#### ✅ Home.tsx（有下拉刷新）
```typescript
const HomeRoute: React.FC = ({ events, user, fetchData }) => {
  usePullToRefresh(fetchData);  // ✅ 有註冊刷新處理
  return <Home events={events} user={user} />;
};
```

#### ✅ MyEvents.tsx（有下拉刷新）
```typescript
const MyEvents: React.FC = ({ events }) => {
    usePullToRefresh(fetchMyGames);  // ✅ 有註冊刷新處理
    // ...
}
```

#### ❌ Search.tsx（修復前，沒有下拉刷新）
```typescript
const SearchPage: React.FC = () => {
    // ❌ 沒有使用 usePullToRefresh
    const [activeTab, setActiveTab] = useState<'all' | 'nearby'>('all');
    // ...
}
```

---

## ✅ 解決方案

### 修改內容

**檔案：** `pages/Search.tsx`

### 1. 新增 import
```typescript
import React, { useState, useEffect, useCallback } from 'react';  // 新增 useCallback
import { usePullToRefresh } from '../contexts/RefreshContext';    // 新增此行
```

### 2. 將 fetchEvents 改為 useCallback
```typescript
// 修改前：
useEffect(() => {
    const fetchEvents = async () => {
        setLoading(true);
        try {
            const data = await api.getEvents();
            setEvents(data);
            setFilteredEvents(data);
        } catch (error) {
            console.error("Failed to fetch events", error);
        } finally {
            setLoading(false);
        }
    };
    fetchEvents();
}, []);

// 修改後：
const fetchEvents = useCallback(async () => {
    setLoading(true);
    try {
        const data = await api.getEvents();
        setEvents(data);
        setFilteredEvents(data);
    } catch (error) {
        console.error("Failed to fetch events", error);
    } finally {
        setLoading(false);
    }
}, []);

// 註冊下拉刷新處理函數
usePullToRefresh(fetchEvents);

useEffect(() => {
    fetchEvents();
}, [fetchEvents]);
```

### 現在用戶可以：

1. ✅ **在搜尋頁面下拉** → 觸發刷新動畫
2. ✅ **重新載入團局列表** → 呼叫 `api.getEvents()`

### 下拉刷新流程：

```
用戶在搜尋頁面下拉
  ↓
PullToRefresh 組件偵測到下拉手勢
  ↓
呼叫 RefreshContext 的 onRefresh()
  ↓
執行 Search.tsx 註冊的 fetchEvents()
  ↓
呼叫 api.getEvents() 取得最新資料
  ↓
更新 events 和 filteredEvents 狀態
  ↓
頁面顯示最新的團局列表
```

## 🔄 下拉刷新系統架構

### 系統組成：

1. **PullToRefresh 組件** (`components/PullToRefresh.tsx`)
   - 偵測用戶的下拉手勢
   - 顯示刷新動畫
   - 呼叫註冊的刷新處理函數

2. **RefreshContext** (`contexts/RefreshContext.tsx`)
   - 管理刷新處理函數的註冊
   - 提供 `usePullToRefresh` hook

3. **各頁面**
   - 使用 `usePullToRefresh(handler)` 註冊刷新邏輯
   - 定義刷新時要執行的操作

### 使用方式：

```typescript
import { usePullToRefresh } from '../contexts/RefreshContext';

const MyPage: React.FC = () => {
    const fetchData = useCallback(async () => {
        // 刷新時要執行的邏輯
        const data = await api.getData();
        setData(data);
    }, []);

    // 註冊下拉刷新處理
    usePullToRefresh(fetchData);

    return <div>...</div>;
};
```

## 📝 注意事項

1. **useCallback 的重要性**
   - 使用 `useCallback` 確保 `fetchEvents` 函數引用穩定
   - 避免無限重新渲染的問題

2. **刷新觸發條件**
   - 只有在頁面滾動位置接近頂部時才能觸發
   - 需要向下拉動超過 80px 的閾值

3. **與其他頁面一致性**
   - 所有列表頁面都應該支援下拉刷新
   - 保持用戶體驗的一致性
